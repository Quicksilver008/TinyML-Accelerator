TinyML-Accelerator Mentor Progress Update

1) Project Goal
- Integrate a TinyML-oriented 4x4 Q5.10 matrix accelerator with PicoRV32 via PCPI custom instruction flow.
- Prove correctness + control handshake in simulation first.
- Compare cycle cost of accelerator path against software baselines on the same CPU family.

2.2 Data/Control Contract Used in Simulation
- rs1 = A matrix base pointer
- rs2 = B matrix base pointer
- C matrix written back by wrapper to memory buffer
- CPU stall/resume handshake proven via:
  - pcpi_wait, pcpi_ready, pcpi_wr, pcpi_rd

2.3 Arithmetic Semantics (Invariant)
- signed16 multiply
- arithmetic right shift by 10
- signed32 accumulation
- low16 wrap/truncate + sign-extension to 32-bit comparison

4) Cycle Comparison Update (Key Improvement)

4.1 What is now compared
- Accelerator path:
  - PicoRV32 PCPI path (ENABLE_PCPI=1, ENABLE_MUL=0)
- Software baseline (no scalar MUL):
  - ENABLE_PCPI=0, ENABLE_MUL=0
  - firmware compiled as rv32i
- Software baseline (scalar MUL enabled):
  - ENABLE_PCPI=0, ENABLE_MUL=1
  - firmware compiled as rv32im

4.2 Latest measured values (identity_x_sequence)
- Script: .\integration\pcpi_demo\scripts\run_cycle_compare.ps1
- Accelerator cycles: 673
- Software no-MUL cycles: 26130
- Software MUL-enabled cycles: 7975
- Speedup (SW no-MUL / accelerator): 38.8262x
- Speedup (SW MUL / accelerator): 11.8499x
- SW MUL benefit (SW no-MUL / SW MUL): 3.2765x

Important technical note:
- For a valid MUL baseline, BOTH are required:
  1) core config ENABLE_MUL=1
  2) firmware compile arch rv32im

5) Robustness / Hygiene Improvements Added
