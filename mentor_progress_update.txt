TinyML-Accelerator Mentor Progress Update
Date: 2026-03-05
Team Branch: omar

1) Project Goal
- Build a TinyML-oriented 4x4 matrix multiplication accelerator integrated with a RISC-V CPU.
- Near-term priority: simulation-first proof of correctness and CPU integration.
- Long-term target: hardware deployment (Pynq-Z2) and measured performance on board.

2) What We Have Completed So Far

2.1 Repository and Core Setup
- Vendored RISC-V reference core:
  - Upstream: https://github.com/srpoyrek/RISC-V
- Vendored PicoRV32 core:
  - Upstream: https://github.com/YosysHQ/picorv32
- Organized project for reproducible simulation flows under:
  - midsem_sim/
  - integration/pcpi_demo/

2.2 Accelerator RTL and Standalone Validation
- Implemented/validated 4x4 Q5.10 systolic-style matrix accelerator RTL flow:
  - PE cell, issue logic, array wiring, top-level controller.
- Standalone mid-sem simulation framework created and passing.

2.3 CPU + Accelerator Integration (PCPI Path)
- Integrated accelerator with PicoRV32 using PCPI custom-instruction interface.
- Added custom instruction decode path and handshake handling:
  - Core stalls during accelerator operation.
  - Core resumes after result-ready handshake.
- Added memory-based matrix flow:
  - rs1 = base address of A matrix buffer
  - rs2 = base address of B matrix buffer
  - C matrix written back to memory buffer by accelerator wrapper.

2.4 Firmware-Driven Input Flow (Not TB Hardcoding)
- Moved matrix input provisioning to firmware (assembly path).
- Firmware copies matrix vectors into RAM, executes custom instruction, then checks result path.
- This allows realistic CPU-driven test behavior instead of direct TB-only injection.

4) Numerical Behavior Model Used
- Fixed-point format: Q5.10
- Verification semantic is RTL-exact:
  - signed16 multiply
  - arithmetic right shift by 10
  - signed32 accumulation
  - low-16 wrap/truncate with sign-extension for memory-word comparison
- This was chosen to match implemented hardware behavior exactly.

5) Current Status Statement (What We Can Claim)
- Accelerator is integrated successfully with PicoRV32 in simulation via PCPI.
- Custom and regular instruction coexistence path is validated.
- Handshake and result-path behavior is verified with targeted metrics.
- Testing is robust at simulation stage (smoke + 8-case regression + handoff scenario).

6) Known Limitations (Transparent)
- Current results are simulation-based; board-level deployment is pending.
- Hardware timing/performance measurements on Pynq-Z2 are not yet collected.
- Native Windows RISC-V toolchain not present; WSL fallback is currently used.
- Full SoC/FPGA top-level integration and bring-up remain next-stage work.
